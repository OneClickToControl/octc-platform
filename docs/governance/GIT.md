# Git governance

Minimum rules for all OneClickToControl LLC repos.

## Branch protection

- `main` protected: requires PR, green status checks (`verify.yml`), 1 approval + CODEOWNERS when touching listed paths.
- Force-push disabled on `main`.
- No direct commits to `main`.

## Conventions

- Conventional Commits.
- Branch naming: `feat/...`, `fix/...`, `chore/...`, `docs/...`, `acp/<id>/...`.
- PRs must use the template in [templates/governance/PULL_REQUEST_TEMPLATE.md](../../templates/governance/PULL_REQUEST_TEMPLATE.md).

## Releases

- SemVer tags per `@1c2c/*` package and per product.
- Changelogs generated by Changesets for JS/TS, manual or `towncrier` for Python.
- Annotated releases also reported to Sentry via `sentry-cli releases new`.

## CODEOWNERS

- Each repo declares CODEOWNERS pointing to real owners.
- Critical paths (manifests, schemas, templates, allowlists, workflows) require platform approval.

## Archived repos

- Legacy repos move to `archived` after 90 days without activity and documented deprecation in [DEPRECATION.md](DEPRECATION.md).
